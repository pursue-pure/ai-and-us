## 敏捷开发原则
1. 优先响应变化而非严格遵循计划，确保交付用户真正需要的价值
2. 团队内部保持透明沟通，每日同步进度与障碍，SM 及时协调解决问题
3. 迭代交付、持续改进，每一个 Sprint 都以可交付的成果为目标
4. 人机协同提效：充分利用 AI 智能体提升编码效率，聚焦核心业务逻辑设计
