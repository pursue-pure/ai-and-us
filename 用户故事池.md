## MVP 核心 User Stories

### US-001: 房间导航

> **As a** 玩家
> **I want** 通过方向命令移动 (go north/south/east/west)
> **So that** 我可以探索游戏世界并发现新场景

**验收标准:**
- [ ] 输入 "go north" 后进入北侧房间
- [ ] 每个房间有唯一描述
- [ ] 无效方向给出友好提示

---

### US-002: 物品拾取

> **As a** 玩家
> **I want** 拾取房间中的物品 (take [物品名])
> **So that** 我可以收集道具用于后续游戏

**验收标准:**
- [ ] 输入 "take sword" 后物品进入背包
- [ ] 房间物品列表更新
- [ ] 不存在的物品给出提示

---

### US-003: 查看状态

> **As a** 玩家
> **I want** 查看当前所在位置和背包 (look/inventory)
> **So that** 我可以了解当前游戏状态

**验收标准:**
- [ ] "look" 显示房间描述和可见物品
- [ ] "inventory" 显示背包内容
- [ ] 空背包有相应提示

---

### US-004: 简单战斗

> **As a** 玩家
> **I want** 与敌人进行回合制战斗 (attack [敌人])
> **So that** 我可以体验游戏的核心挑战

**验收标准:**
- [ ] 输入 "attack goblin" 进入战斗
- [ ] 显示战斗结果 (伤害/血量)
- [ ] 敌人血量归零后掉落物品

---

### US-005: 游戏存档

> **As a** 玩家
> **I want** 保存和加载游戏进度 (save/load)
> **So that** 我可以中断后继续游戏

**验收标准:**
- [ ] "save" 命令保存当前状态到 JSON
- [ ] "load" 命令恢复游戏状态
- [ ] 存档文件可手动查看

---

## 开发原则

本项目的 MVP 遵循以下原则：

- ✅ 纯控制台交互，无 GUI
- ✅ 无网络功能
- ✅ 简单命令解析（关键词匹配）
