# 用户故事池 - MUD 洞穴探险游戏 Sprint 1

## MVP 核心 User Stories

### US-001: 房间导航 ✅

> **As a** 玩家
> **I want** 通过方向命令移动 
> **So that** 我可以探索洞穴的各个房间

**验收标准:**
- [x] 输入 "go north" 或 "n" 后向北移动
- [x] 每个房间有独特的环境描述
- [x] 无效方向给出友好提示
- [x] 提示前往 BOSS 的方向

---

### US-002: 物品拾取 ✅

> **As a** 玩家
> **I want** 拾取房间中的物品
> **So that** 我可以收集道具提升实力

**验收标准:**
- [x] 输入 "take 铁剑" 后物品进入背包
- [x] 支持 `take<铁剑>`、`take铁剑` 等常见输入格式
- [x] 必须先输入 `look` 搜索才能发现物品
- [x] 房间物品列表更新
- [x] 不存在的物品给出提示

---

### US-003: 查看状态 ✅

> **As a** 玩家
> **I want** 查看当前所在位置和背包
> **So that** 我可以了解当前游戏状态

**验收标准:**
- [x] 进入房间自动显示房间描述（不自动显示物品）
- [x] "look" 用于搜索并发现物品
- [x] "inventory" 显示背包内容和物品效果
- [x] 空背包有相应提示

---

### US-004: 战斗系统 ✅

> **As a** 玩家
> **I want** 与敌人进行回合制战斗
> **So that** 我可以体验游戏的战斗玩法

**验收标准:**
- [x] 输入 "attack" 攻击当前房间的敌人
- [x] 显示战斗过程（玩家伤害、敌人反击）
- [x] 敌人血量归零后玩家获得经验
- [x] 玩家血量归零时死亡
- [x] 击败 BOSS 后游戏胜利

---

### US-005: 死亡复活 ✅

> **As a** 玩家
> **I want** 死亡后在上一房间复活 (respawn)
> **So that** 我可以继续游戏而不必从头开始

**验收标准:**
- [x] 战斗失败后玩家 HP 归零
- [x] 输入 "respawn" 复活
- [x] 复活地点为检查点
- [x] 复活时恢复一半 HP
- [x] 存档与读档可恢复检查点信息（位置与时间）

---

### US-006: 游戏存档 ⏳

> **As a** 玩家
> **I want** 保存和加载游戏进度
> **So that** 我可以中断后继续游戏

**验收标准:**
- [x] "save" 命令保存当前状态到 JSON
- [x] "load" 命令恢复游戏状态
- [ ] 存档完整恢复房间状态（敌人与物品）
- [x] 存档文件可手动查看

---

### US-007: 升级系统 ✅

> **As a** 玩家
> **I want** 击败怪物后获得经验升级
> **So that** 我可以变得更强来挑战 BOSS

**验收标准:**
- [x] 击败怪物获得经验值
- [x] 达到经验阈值自动升级
- [x] 升级时最大生命值 HP +20，且生命值回满，攻击 +5
- [x] 显示当前等级和经验进度

---

## 游戏设计文档

### 地图流程

```
🕳️ 洞穴入口 → 📍 阴暗走廊 → 💀 哥布林营地 → 💎 宝藏室 → 👹 兽人大厅 → ⚔️ 武器库 → 🐉 BOSS 巢穴
   安全          安全 + 物品      小怪 1        安全 + 宝物     小怪 2        安全 + 神器    最终 BOSS
```

### 玩家属性

| 属性 | 初始值 | 升级奖励 |
|------|--------|----------|
| HP | 50 | +20 |
| 攻击 | 10 | +5 |
| 等级 | 1 | +1 |

| 🐗 兽人战士 | 40 | 10 | 60 | 兽人大厅 |

## 开发原则

本项目的 MVP 遵循以下原则：

- ✅ 纯控制台交互，无 GUI
- ✅ 无网络功能
- ✅ 简单命令解析（关键词匹配）
- ✅ 回合制战斗
- ✅ 单机单玩家
